# E-Commerce Product Search (TypeScript)

TypeScript implementation of the gallery example using `@superlinked/sie-sdk`, Express, and JSON-based embeddings.

Pipeline:

1. `extract()` pulls structured attributes from product descriptions
2. `encode()` embeds the catalog into dense vectors
3. `search.ts` retrieves by cosine similarity and reranks with `score()`
4. `server.ts` serves the shared UI and REST endpoints

## Files

- `src/ingest.ts` builds `data-ts/embeddings.json` and `data-ts/metadata.json`
- `src/search.ts` runs CLI demo searches against the saved index
- `src/server.ts` serves `/`, `/api/search`, `/api/categories`, and `/api/stats`
- `package.json` contains the TypeScript app dependencies and scripts

## Requirements

- Node.js 22+
- `pnpm`
- A reachable SIE server. Local Docker is the default:
  ```bash
  docker run -p 8080:8080 ghcr.io/superlinked/sie-server:latest-cpu-default
  ```
  Or a managed SIE cluster via `SIE_CLUSTER_URL` and `SIE_API_KEY` env vars.
- Shared dataset file `../data/products.json` (generated by `python data/fetch_dataset.py` from the example root)

## Configuration

This implementation reads `../config.yaml`. The defaults target a local SIE server at `http://localhost:8080`.

See the [top-level README](../README.md#customize) for the full config.

To point at a managed cluster instead:

```bash
export SIE_CLUSTER_URL="https://your-cluster-url"
export SIE_API_KEY="your-api-key"
```

## Run It

From `examples/ecommerce-product-search/`:

```bash
pip install -r python/requirements.txt
python data/fetch_dataset.py
cd typescript
pnpm install
pnpm ingest
pnpm search
pnpm dev
```

Open `http://localhost:3000`.

This also serves the shared browser UI, so you can type a query, choose a category, and explore results interactively without using the API directly.

Generated files:

- `../data/products.json`
- `data-ts/embeddings.json`
- `data-ts/metadata.json`

If you want the compiled server path instead of the dev server:

```bash
pnpm build
pnpm start
```

## API

### `GET /api/search`

Parameters:

- `q` required query string
- `category` optional category filter
- `brand` optional extracted brand filter

Response fields:

- `query`
- `filters`
- `results[].id`
- `results[].title`
- `results[].description`
- `results[].category`
- `results[].price`
- `results[].rating`
- `results[].image`
- `results[].features`
- `results[].attributes`
- `results[].scores.vector`
- `results[].scores.rerank`

### `GET /api/categories`

Returns category counts as `[name, count]` tuples.

Example:

```json
[
  ["AMAZON FASHION", 613],
  ["Amazon Home", 549]
]
```

### `GET /api/stats`

Returns:

```json
{
  "total_products": 3000,
  "embedding_dims": 1024,
  "categories": 7,
  "models": {
    "embedding": "NovaSearch/stella_en_400M_v5",
    "reranker": "BAAI/bge-reranker-v2-m3",
    "extractor": "urchade/gliner_multi-v2.1"
  }
}
```

## Notes

- This path reads the shared dataset from `../data/products.json`, so generate that file first with `python data/fetch_dataset.py` from the example root.
- The TypeScript implementation stores embeddings in JSON and normalizes them in memory during search.
- If reranking fails, the search path falls back to vector-ranked results.

## Troubleshooting

### Missing dataset files

From `examples/ecommerce-product-search/`, run:

```bash
python data/fetch_dataset.py
```

### Missing TypeScript index files

Run:

```bash
cd typescript
pnpm ingest
```

### Cannot connect to the cluster

- Check `cluster.url` and `cluster.api_key` in `../config.yaml`
- Or set `SIE_CLUSTER_URL` and `SIE_API_KEY`
- Verify the cluster can serve the configured models
